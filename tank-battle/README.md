# STEEL STORM — Tank Battle

A top-down tank battle in **TypeScript** and HTML5 Canvas. Defend your HQ, shred
destructible cover, and clear waves of enemy armor across three sectors. Think
classic *Battle City* (1985), rebuilt from scratch with zero assets — every tank,
tile and sound is generated by code at runtime.

## Run it

```bash
cd tank-battle
npm install
npm run dev          # http://localhost:5174
```

Build a standalone bundle:

```bash
npm run build        # type-checks (tsc --noEmit) then bundles to dist/
npm run preview      # serve the production build
```

## Controls

| Key | Action |
| --- | --- |
| `WASD` / Arrows | drive (4-directional, auto-aligns to corridors) |
| `Space` / `J` | fire |
| `P` | pause |
| `M` | mute |
| `Enter` | start / restart |

## What's inside

- **4-directional grid movement** — tanks snap to lanes on turn; AABB collide-and-slide
  physics sub-stepped against a 26×26 tile grid.
- **Destructible terrain** — brick walls chip away per-tile, steel blocks everything
  (until a level-3 player punches through it). Water blocks tanks but not shells;
  bushes hide tanks under their canopy.
- **Four enemy types** — Basic, Fast (cyan), Power (pink, steel-piercing shells) and
  Armor (green, four hits). Each runs a goal-seeking AI that heads for your HQ or your
  tank, re-picks a route when blocked, and fires harder when it has a shot lined up.
- **Player upgrades** — collect **★** stars to level up (faster fire, twin shells,
  then steel-cracking rounds). **⛨ Shield**, **❤ 1UP** and **✸ Bomb** powerups drop
  from defeated enemies.
- **Three hand-authored sectors** — maps are plain ASCII art in `src/levels.ts`
  (` ` empty · `B` brick · `S` steel · `W` water · `T` bush). Edit and the layout
  changes — same text-map philosophy as the rest of this monorepo.
- **Synthesized audio** — every sound (shoot, explosion, powerup, jingles) is
  generated live with the WebAudio API; no audio files.
- **State machine** — title → ready → playing → cleared → victory, with pause and
  game-over. High score persists in `localStorage`.

## Project layout

```
tank-battle/
  index.html
  src/
    main.ts        canvas + game loop
    game.ts        phase state machine, score, lives, hi-score
    world.ts       tile grid, collision, spawning, firing, powerups
    entities.ts    Tank, Bullet, Explosion, PowerUp
    ai.ts          enemy goal-seeking AI
    render.ts      canvas drawing (field, tanks, HUD, overlays)
    levels.ts      ASCII sectors + terrain parser
    input.ts       keyboard input (movement stack + fire)
    audio.ts       WebAudio synth SFX
    types.ts       shared types & enums
    constants.ts   tunables, palette, unit configs
```

No runtime dependencies — TypeScript + Vite (dev only). Pure Canvas 2D.
