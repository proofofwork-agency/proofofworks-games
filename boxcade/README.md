# Boxcade

**Build. Play. Together.** — a free, open browser game platform and engine: friendly blocky
worlds with AAA-style lighting. Type one command, get a portal of multiplayer games, and create
your own in ~25 lines of TypeScript.

```bash
npm install
npm run dev        # client (http://localhost:5173) + multiplayer room server, one command
```

No game assets, no accounts, no build pipeline for creators. Every model, face, sound and sky
in Boxcade is generated by code at runtime.

## What Boxcade is (and isn't) — honest scope

Boxcade is a **genre engine + platform seed**, not a general-purpose engine. Within its genre
family — blocky platformers/obbies, voxel sandboxes, arena shooters, CTF, battle royale,
adventure maps — a creator gets a complete multiplayer-flavored game from one TypeScript file,
and can extend the engine itself through registries, events and systems (see
[`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md)).

Current ceilings, so you know before you build:

- **Multiplayer is presence-only**: players see each other and chat, but game state (combat,
  moving parts, scores) is local — server-authoritative replication is the #1 roadmap item.
- **Games are code in this repo**: adding a game = adding a file to `src/games/`. There is no
  runtime upload/publish path yet (the map editor covers maps, not full game logic).
- **Physics is axis-aligned boxes** — no slopes, rotated colliders, vehicles or ragdolls.
- **Procedural-only content** — no model/asset import (that's also the zero-pipeline appeal).
- Character-controller games are the sweet spot; top-down/2D/card genres aren't served.
- localStorage persistence only; no accounts, saves or leaderboards; no touch controls yet.

## Why these design choices (the research)

Boxcade's design is reverse-engineered from why the biggest blocky game platforms work.

### The social-platform playbook

| Insight | Boxcade implementation |
| --- | --- |
| Third-person **WASD + Space**, drag-to-orbit camera, scroll zoom, shift-lock, **R to reset** | The default control scheme (`engine/camera.ts`, `engine/input.ts`) |
| The **obby** (obstacle course) is blocky gaming's foundational genre | *Sky Obby* — 15 stages, checkpoints, kill bricks, spinners, ice, bounce pads |
| Blocky avatars with simple faces — simplicity is identity, not a limitation | Procedural rounded-box avatar, canvas-drawn smile, per-player shirt colors |
| Social presence drives retention: friends, chat, name tags | WebSocket rooms, interpolated avatars, chat bubbles, join/leave toasts |
| Creation must be radically easy | `defineGame({...})` SDK — a complete game is one small file |
| Platform model: a **portal of games**, not one game | Hash-routed portal with game cards; adding a game = 1 import |
| Gamenomics: play → earn → spend keeps creators and players invested | Session coin economy (`ctx.award`) as the hook for a future creator economy |

### The voxel-sandbox playbook

| Insight | Boxcade implementation |
| --- | --- |
| Place/break 1m voxels = infinite creativity from one verb pair | *Voxel Island*: LMB break / RMB place, hotbar 1–8 |
| Pointer-lock first person | `camera: 'fp'` mode |
| **Per-vertex ambient occlusion** + face shading make flat colors read as 3D | Classic 3-neighbor corner AO with anisotropy-corrected quad flipping (`engine/voxel.ts`) |
| 16×16 chunks, hidden-face culling | Chunked meshing, per-chunk rebuild on edit |
| Worlds belong to players | Download any voxel world as JSON (pause menu) |

### The AAA-lighting playbook (the "looks expensive" part)

- **ACES filmic tone mapping** — the industry-standard filmic curve
- **Bloom** — via three.js's `UnrealBloomPass` addon (lava, neon and sun glow)
- **GTAO** ground-truth ambient occlusion pass
- **SSR** screen-space ray-traced reflections on shiny materials (ice, metal, gold, glass, water)
- PCF soft shadows snapped to texels (no shimmer), following the player
- Sky-tinted fill light + boosted hemisphere as fake GI, so shadows never crush to black
- Procedural sky shader: atmosphere gradient, sun disc + halo, FBM clouds, night stars

The bet: **premium light on friendly blocky shapes** beats realistic geometry. Low-poly worlds
+ a modern post stack = runs everywhere, looks expensive.

## Combat, weapons & bots

`GameDef.combat` switches on the arsenal — seven tournament-style archetypes, all plain data
(`WEAPONS` in the SDK), every number tunable:

| Weapon | Type | Notes |
| --- | --- | --- |
| 🔫 Sidearm | hitscan | starting pistol |
| ⚡ Shock Lance | hitscan beam | the long-range duel weapon |
| 🟢 Pulse Blaster | projectile | fast plasma spam |
| 🌀 Minigun | hitscan | high rate, wide spread |
| 💥 Flak Scattergun | 8-pellet projectile | shotgun arcs with gravity |
| 🚀 Rocket Launcher | projectile + splash | knockback — rocket jumps work |
| 🎯 Sniper Rifle | hitscan | right-click zoom, 2× headshots |

Ammo is limited (the sidearm never runs dry): weapons track per-entity ammo pools, the HUD shows
a live ammo counter + per-slot counts, and maps place **weapon pads**, **ammo crates** and
health packs (`w.weaponSpawn / w.ammoSpawn / w.healthPack`) that respawn on timers. Everyone you
hit shows it: overhead life bars shrink, bodies flash red, and damage numbers float up.

Bots (`ctx.spawnBot({ name, team, skill, spawns })`) run the same character physics as players:
they pick the right gun for the range, lead their projectiles, fire in bursts, hunt whoever shot
them, loot weapon/ammo/health pickups when low, hop low walls, strafe in fights, and play
CTF (carry → return, recover dropped flags, chase thieves). Movement is tunable per game or per
map: `physics: { gravity, jumpVel, walkSpeed }` in code, or `@gravity / @jump / @speed` in any
text map — Facing Towers runs at low gravity for that floaty space feel.

Honest scope note: combat is locally authoritative (your shots hurt bots in your session).
Remote humans are visible but not yet damageable — server-authoritative PvP is on the roadmap.

## Gamenomics (the platform loop, miniaturized)

The classic platform economy runs: play → earn → spend on your avatar → express yourself →
play more. Boxcade implements the same flywheel locally:

- **Earn Bolts**: coins +1 each · kills +10 (headshot +15) · flag captures +50 · match wins +150 ·
  any win pad +25 · daily login bonus +100
- **Spend Bolts**: the portal **Shop** sells shirts and particle trails (sparkle/fire/rainbow);
  equipped items persist and show in every game
- Wallet + inventory live in `localStorage` (`engine/economy.ts`) — swapping in a real backend
  later means replacing one small class

## The Map Editor

Open **🗺 Map Editor** on the portal (`#/editor`): a visual painter for the text-map format.
Paint tiles with the palette, stack floors, resize the grid, tune **gravity / jump power / walk
speed / lighting** with form fields, then **▶ Test play** drops you straight into your map.
Download as `.txt`, copy to clipboard, or paste any existing map in — the textarea at the bottom
*is* the map file, synced live in both directions. Templates included (Castle Run, Facing Towers).

Open two browser tabs to see multiplayer: avatars, name tags and chat sync through the room
server. No server? Games silently run solo.

## Make a game in 2 minutes

Create `src/games/my-game.ts`:

```ts
import { defineGame, v3, behaviors, colors } from '../sdk'

export default defineGame({
  meta: {
    id: 'my-game', name: 'My Game', emoji: '🚀', genre: 'Adventure',
    blurb: 'My first Boxcade game.',
    gradient: 'linear-gradient(135deg, #f953c6, #b91d73)',
  },
  build(w) {
    w.lighting('goldenHour')                 // 'noon' | 'morning' | 'goldenHour' | 'night'
    w.spawn(v3(0, 4, 0))
    w.add({ at: v3(0, 0, 0), size: v3(24, 2, 24), color: colors.grass, material: 'grass' })
    w.add({                                   // a spinning platform
      at: v3(8, 3, 0), size: v3(4, 1, 4), color: colors.sky,
      behavior: behaviors.spin(1.5),
    })
    w.lava(v3(0, -10, 0), v3(200, 2, 200))    // kill floor
    w.coin(v3(2, 3, 2))
    w.bouncePad(v3(-6, 1.4, -6))
    w.winPad(v3(8, 5, 0))
  },
  onStart(ctx) { ctx.hud.toast('Welcome!') },
  onTick(ctx, dt) { /* per-frame logic */ },
})
```

Register it in `src/games/index.ts` (one import + one array entry). It appears on the portal
with multiplayer, chat, physics, HUD and sound already working.

## Draw maps in plain text files — no code

Levels can be ASCII art. One character = one tile, `---` starts the next floor.
This is a complete playable map:

```text
@lighting goldenHour

..........333..........
..C....1..3W3..1....C..
.######.##333##.######.
.#....#.#.....#.#....#.
.#.S..#L#..K..#L#..B.#.
.#....#.#.....#.#....#.
.######.#######.######.
```

| Char | Meaning | Char | Meaning |
| --- | --- | --- | --- |
| `#` | stone floor | `G` | grass |
| `O` | planks | `X` | brick |
| `I` | ice (slippery) | `N` | glowing neon |
| `1`–`9` | stone column that tall | `L` | lava (kills) |
| `B` | bounce pad | `C` | tile + coin |
| `K` | checkpoint (auto-numbered) | `T` | tile + tree |
| `S` | spawn | `W` | golden win pad |
| `H` | health pack (+35 hp) | `A` | ammo crate (restocks weapons) |
| `M` | metal deck (mirror-shiny) | `F` / `f` | red / blue flag stand (CTF) |
| `r` / `b` | red / blue team spawn | `.` | void (falling = death) |

Directives: `@lighting` (incl. `space`), `@cell`, `@layerstep`, `@killy`, and movement tuning:
`@gravity`, `@jump`, `@speed`. Comment lines start with `//`.

**Workflow** (this is the "map editor"): run `npm run dev`, open your game, then edit the
`.txt` file in any editor and save — Vite hot-reloads you into the new layout in under a
second. See `src/maps/castle.txt` + `src/games/castle-run.ts` (3 lines to wire a map in:
`import map from '../maps/castle.txt?raw'` → `buildTextMap(w, map)`). Text and SDK calls mix
freely, so decorations, moving platforms and scripted logic still work on top.

### SDK cheat sheet

- **Parts**: `w.add({ at, size, color, material, collide, bounce, behavior, onTouch })`
  - materials: `plastic · grass · wood · stone · ice (slippery!) · neon · lava · gold · glass · metal`
- **Behaviors**: `behaviors.spin(speed)` · `patrol(offset, period)` · `orbit(center, r, period)` · `bob(amp, period)`
- **Prefabs**: `w.checkpoint() · w.lava() · w.coin() · w.winPad() · w.bouncePad() · w.tree() · w.cloud() · w.spinnerHazard() · w.label() · w.weaponSpawn() · w.ammoSpawn() · w.healthPack()`
- **Voxels**: `w.voxelIsland({ seed, size })` + `camera: 'fp'` = full build mode
- **Context** (in `onStart`/`onTick`/`onTouch`): `ctx.player` (kill/respawn/teleport/launch/setCheckpoint),
  `ctx.hud` (set/toast/big), `ctx.award(n)`, `ctx.celebrate()`, `ctx.systemChat()`, `ctx.addPart()`

### Composability — every layer is extensible

The engine is built from swappable subsystems behind one SDK
(see [`docs/ARCHITECTURE.md`](./docs/ARCHITECTURE.md) for the full map):

- **Registries** — add content once, use it anywhere by name:
  `registerWeapon(def)` · `registerMaterial(kind, factory, { reflective })` ·
  `registerTile(char, handler)` (text maps + editor) · `registerSkyPreset(name, preset)`
- **Events** — `ctx.events.on('combat:kill' | 'combat:pickup' | 'self:damage' | …)`,
  plus your own namespaced events; the engine publishes, games observe (no coupling)
- **Game systems** — `systems: [dayNight, killStreaks]` in `defineGame`: reusable
  logic units with `init/update/dispose` lifecycles instead of one giant `onTick`
- **`ctx.engine`** — the power API: live access to `renderer`, `parts`, `voxels`,
  `combat`, `fx`, `net`, `input`, `player`, `audio` for advanced games
- **Data-driven defs** — weapons, parts, behaviors, physics and whole maps are
  plain data; clone a def and tweak numbers to make variants

## Controls

| | Obby / third person | Voxel Island / first person |
| --- | --- | --- |
| Move / jump | `WASD` + `Space` (hold Space to bunny-hop) | same |
| Camera | drag with either mouse button (pointer auto-captures for full 360° spins) · scroll zoom · `Shift` toggles mouse-look | click to capture mouse |
| Act | `R` reset to checkpoint | left-click break · right-click place · `1–8`/scroll blocks |
| Social | `/` or `Enter` chat · `M` mute · `Esc` pause | same |

## Architecture

```
src/
  engine/     the engine — zero game knowledge
    renderer.ts   three.js + modern post stack (ACES, bloom, GTAO, SSR, soft shadows)
    sky.ts        procedural sky shader (atmosphere, sun, FBM clouds, stars) + presets
    physics.ts    AABB collide-and-slide controller: coyote time, step-up, ice,
                  bounce pads, water buoyancy, moving-platform carry, substepping
    world.ts      Parts: rounded boxes, materials, behaviors, triggers
    voxel.ts      chunked voxels: culled meshing, vertex AO, DDA raycast, RLE export
    avatar.ts     procedural blocky character: walk/jump animation, face, name tag, chat bubble
    camera.ts     orbit rig (third person) + pointer-lock rig (first person), camera collision
    network.ts    WebSocket client, 12Hz send, 120ms interpolation buffer, offline fallback
    audio.ts      WebAudio synth — every sound is code (jump, coin, fanfare, crunch…)
    fx.ts         one-draw-call particle pool (confetti, sparkles, debris)
    input.ts, math.ts
  sdk/          the creator-facing API (defineGame + types + prefab vocabulary)
  runtime/      glues a GameDef to the engine: HUD, chat, build mode, pause, game loop
  games/        squadfall.ts · facing-towers.ts · sky-obby.ts · voxel-island.ts · castle-run.ts ·
                starter.ts · index.ts (registry)
server/         room server: rooms per game id, 15Hz state fan-out, chat rate limiting
```

Total: ~6,500 lines of TypeScript, two runtime dependencies (`three`, `ws`), ~185KB gzipped.

## Commands

```bash
npm run dev      # client + room server together
npm run client   # vite only (games auto-fallback to solo mode)

```
